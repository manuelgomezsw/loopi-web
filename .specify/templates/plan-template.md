## Implementation Phases

### Phase 0: Research & Verification

- Review existing services in `core/services/` that this feature interacts with.
- Confirm API endpoint contract (`environment.apiUrl` base).
- Confirm no naming conflicts with existing components or routes.

### Phase 1: Models & Service Layer

1. Add new interfaces/types to `src/app/core/models/<domain>.model.ts` (or create new file).
2. Export from `src/app/core/models/index.ts`.
3. Implement or extend service in `src/app/core/services/<name>.service.ts`:
   - State as `signal<T>()`, exposed as `.asReadonly()`.
   - HTTP calls via `inject(HttpClient)`.
   - Errors handled with `catchError`; do not throw to components.

### Phase 2: Component Layer

1. Create new standalone component(s) in `src/app/features/<feature>/<screen>/`.
2. Use `inject()` to consume services.
3. Derive view state with `computed()`.
4. Style with Tailwind CSS v4 utility classes.
5. Wire into route config via `loadComponent` (or `loadChildren` for sub-routes).

### Phase 3: Tests (TDD ‚Äî write spec before implementation)


### Phase 4: Integration & CI Gate

- `npm run build` ‚Äî must succeed with no new TypeScript errors.
